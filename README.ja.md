# Nemuri

[English](README.md) · [简体中文](README.zh-CN.md) · [繁體中文](README.zh-TW.md) · **日本語** · [한국어](README.ko.md)

**Nemuri は、Claude Code や Codex が動いているあいだ Mac を眠らせない macOS のメニューバーアプリです — フタを閉じたままでも。仕事が終わったら、また自分でスリープに戻します。**

エージェントに長いタスクを渡して、ノートを閉じ、そのまま離席する。フタを閉じた瞬間に固まらず、処理はそのまま走り続けます。最後のエージェントが終わったら、Mac は自分でスリープへ — 1 時間前に終わった仕事のために、閉じた MacBook がカバンの中で熱を持ち続けることもありません。

![Nemuri のメニューバーパネル：2 つのエージェントを追跡中 — 片方は実行中、片方はあなた待ち — 終わったら自動でスリープ](docs/panel-pro.png)

## ダウンロード

- **DMG** — [署名・公証済みの最新リリースをダウンロード](https://github.com/syfssb/nemuri/releases/latest)（macOS 13+、Apple Silicon & Intel）

Mac App Store にはありません — 本物のクラムシェルモードに必要な特権ヘルパーを、サンドボックスが禁止しているためです。製品の詳しい紹介は <https://nemuri.app/ja/> をご覧ください。

## できること

- **フタを閉じて、外部ディスプレイなしでも走り続ける。** `caffeinate` だけでなく、ファームウェアレベルの `disablesleep` フラグを使った本物のクラムシェルモードです。
- **「実行中」「あなた待ち」「完了」を見分ける。** エージェントが実際に動いているあいだは起こしておき、入力待ちで止まった瞬間に知らせ、最後の 1 つが終わって数分後にスリープを戻します。*(Pro)*
- **セッションパネル。** どのエージェントが、どのプロジェクトで、何分走っているか。*(Pro)*
- **Mac を起こしっぱなしにしない。** クラッシュ・再起動・終了・アンインストール — `disablesleep` は必ず `0` に戻ります。4 つの復旧経路、それぞれに自動テスト付き。
- **静かに動く。** アカウントなし、テレメトリなし、バックグラウンド通信なし。

## このリポジトリを公開している理由

Nemuri は、**root ヘルパー**の承認をあなたにお願いします — root で動き、システムの `disablesleep` フラグを切り替えるバックグラウンドプロセスです。これは大きなお願いです。このリポジトリは Nemuri の信頼にかかわる部分で、私たちの言葉を信じる代わりに、あなた自身が読めるように公開しています。

**Free と Pro は、このリポジトリの同じコードを動かしています。** Pro を買っても買わなくても、あなたの信頼を必要とする部分は、ここに公開されているこの部分です。

| このリポジトリの中身 | 何か |
|---|---|
| `Sources/Helper` | root LaunchDaemon — root で動く唯一のコンポーネント。 |
| `Sources/Shared` | アプリとヘルパーのあいだの XPC 契約 — ヘルパーの攻撃面のすべて。 |
| `Sources/Core` | `pmset`／センチネル／バッテリーのプリミティブ（詰まないことの保証）、エージェント設定を書き換えるインストーラー、そしてローカル IPC プロトコル。 |
| `Sources/HookBridge` | `aw-hook` / `aw-codex` — あなたの `~/.claude/settings.json` と `~/.codex/config.toml` に書き込まれる、ごく小さなバイナリ。 |

**ここにないもの：** 検出エンジン、ステートマシン、オフラインのライセンス検証、そして SwiftUI アプリ。これらはクローズドソースで、Nemuri Pro が売っているのはまさにそこです。このツリーからアプリ全体をビルドすることはできませんし、DMG の中のバイナリも、このソースだけから作られたものではありません。これは実際の制約で、ぼかさずはっきり書いておきます。

## ここで検証できること

- **root の攻撃面が最小。** ヘルパーが公開する XPC メソッドは 3 つ — `setSleepDisabled(Bool)`、`currentState()`、`ping()` — で、あとは `/usr/bin/pmset` を実行するだけ。任意コマンドの実行なし、センチネル以外への書き込みなし、ネットワークなし。
- **詰まない。** アプリがクラッシュ・kill・アンインストールされても、`pmset disablesleep` は `0` に戻ります。ウォッチドッグ、60 秒ごとのセンチネル自己チェック、起動時の復旧 — すべて `Sources/Helper/main.swift` にあります。
- **隠れたネットワークがない。** このツリーを `URLSession`、`Network`、`socket(`、`connect(` で grep してみてください — ソケットは `AF_UNIX`（ローカル IPC）だけで、root ヘルパーがインターネットに出ることはありません。
- **設定はバックアップされ、元に戻せる。** `~/.claude/settings.json` と `~/.codex/config.toml` を書き換えているのは `Sources/Core/Installer.swift` です。書き込む前にバックアップを取り、完全に復元できます。

## プライバシー

Nemuri は、日常の動作ではネットワークをまったく使いません。インターネットに触れるのは、あなたが操作したときだけ：「アップデートを確認…」の裏側での更新チェックと、Pro を購入したときにこの Mac をライセンスに紐付ける一度きりの認証です。そのあとのライセンス検証は、内蔵の Ed25519 公開鍵に対して完全にローカルで行います — テレメトリなし、バックグラウンドの再接続なし。デバイス紐付けより前に購入したライセンスは、その一度すら通信せず、完全にオフラインで認証します。

## 自分でビルドする

```bash
swift build   # root ヘルパー、XPC 契約、core プリミティブ、2 つの hook ブリッジをビルドします
```

macOS 13+ と Swift 5.9+ が必要です。このツリーは単体でビルドできます — クローズドソースの依存も、サードパーティのパッケージも、ビルド時のネットワークアクセスもありません。まず root のコンポーネントから読むなら、`Sources/Helper/main.swift`（意図的に短く書いてあります）と `Sources/Shared/AwakeShared.swift`（誰がヘルパーに命令してよいかを決める XPC の requirement）から始めてください。
