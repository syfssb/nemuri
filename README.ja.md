# Nemuri — open core

[English](README.md) · [简体中文](README.zh-CN.md) · [繁體中文](README.zh-TW.md) · **日本語** · [한국어](README.ko.md)

### Free — 手動のキープアウェイクのスイッチ

![Nemuri Free パネル：手動のキープアウェイク、agent 検知なし](docs/panel-free.png)

オンにすれば、蓋を閉じても Mac は起きたままです。検知も自動化もありません——自分でオフにします。
無料版がやるのはここまでです。

### Pro — 検知の自動化

![Nemuri Pro パネル：2 つの agent を見守り中、終わったら自動でスリープ](docs/panel-pro.png)

Pro は agent を見守ります。実際に動いている間だけ Mac を起こしておき、確認待ちになったら知らせ、
終わったら Mac をスリープに戻します。**このパネルの背後にある検知エンジンはクローズドソースで、
このリポジトリには含まれていません。**
**[Nemuri](https://nemuri.app/ja/) のうち、root で動く部分、あなたの設定に触れる部分、外部に通信しうる部分 — 自分で確かめられるように公開しています。**

Nemuri は macOS のメニューバーツールです。AI エージェント（Claude Code、Codex）が働いているあいだ Mac を起こしておき — フタを閉じたままでも — 仕事が終わったらスリープへ戻します。

そのために Nemuri は、**root ヘルパー**の承認をあなたにお願いします。これは大きなお願いです。このリポジトリは、私たちの言葉を鵜呑みにしなくて済むように存在します。

## 🔍 ここにあるもの（と、ないもの）

**このリポジトリはアプリ全体ではありません。** 信頼にかかわる部分だけを集めた部分集合です。

| このリポジトリの中身 | なぜここにあるか |
|---|---|
| `Sources/Helper` | **root LaunchDaemon**。root で動く唯一のコンポーネントです。 |
| `Sources/Shared` | アプリとヘルパーのあいだの XPC 契約 — ヘルパーの攻撃面のすべてです。 |
| `Sources/Core` | `pmset`／センチネル／バッテリーのプリミティブ（詰まないことの保証）、エージェント設定を書き換えるインストーラー、そしてローカル IPC プロトコル。 |
| `Sources/HookBridge` | `aw-hook` / `aw-codex` — あなたの `~/.claude/settings.json` と `~/.codex/config.toml` に書き込まれる、ごく小さなバイナリです。 |

**このリポジトリにないもの：** 検出エンジン（プロセス／セッション／rollout のヒューリスティック）、ステートマシン、オフラインのライセンス署名検証、そして SwiftUI アプリ。これらはクローズドソースです — Nemuri Pro が売っているのは、まさにそこです。

ですから、これで何が手に入るのかは正確に言います。あなたが監査できるのは、**root で何が動くか**、**設定に何が書き込まれるか**、**インターネットへソケットを開くコードがあるか** です。このリポジトリから Nemuri アプリ全体をビルドすることはできませんし、署名済み DMG に入っているバイナリも、このソースツリーだけから作られたものではありません。

これは実際の制約です。ぼかすより、はっきり書くほうを選びます。

## ✅ ここで検証できること

- **root の攻撃面が最小であること。** ヘルパーが公開する XPC メソッドはちょうど 3 つ — `setSleepDisabled(Bool)`、`currentState()`、`ping()` — で、あとは `/usr/bin/pmset` を実行するだけ。それ以外はありません。任意コマンドの実行なし、センチネル以外へのファイル書き込みなし、ネットワークなし。
- **スリープ禁止のまま詰まないこと。** アプリがクラッシュしても、kill されても、アンインストールされても、`pmset disablesleep` は必ず `0` に戻らなければなりません。ウォッチドッグ、60 秒ごとのセンチネル自己チェック、起動時の復旧 — すべて `Sources/Helper/main.swift` にあります。
- **ネットワークがゼロであること。** このツリーを `URLSession`、`Network`、`socket(`、`connect(` で grep してみてください。ソケットは `AF_UNIX`（ローカル IPC）だけです。ヘルパーがインターネットと話すことはなく、ライセンス認証も同じです — Nemuri Pro のロックを外すのはオフラインの Ed25519 署名ファイルで、検証はローカル。機内モードでも動きます。
- **設定はバックアップされ、元に戻せること。** `~/.claude/settings.json` と `~/.codex/config.toml` を書き換えているのは `Sources/Core/Installer.swift` です。書き込む前にバックアップを取り、完全に復元できます。

## 🛠 自分でビルドして、自分で監査する

```bash
swift build            # ヘルパー、hook ブリッジ、core をビルドします
```

macOS 13+ と Swift 5.9+ が必要です。このツリーは単体でビルドできます — クローズドソースの依存はなく、ビルド時のネットワークアクセスもありません。

まず root のコンポーネントから読むなら、`Sources/Helper/main.swift`（意図的に短く書いてあります）と `Sources/Shared/AwakeShared.swift`（誰がヘルパーに命令してよいかを決める XPC の requirement 文字列）から始めてください。

## 📦 Nemuri を入手する

リリース（署名・公証済みの DMG）は、このリポジトリの [Releases](https://github.com/syfssb/nemuri/releases) ページで配布します。v1.0 が出たあとは Homebrew からも入れられます。手動のキープアウェイクのスイッチは無料で使えます。検出の自動化は買い切りの有料アップグレードです。詳しくは <https://nemuri.app/ja/> をご覧ください。
