# Nemuri

[English](README.md) · [简体中文](README.zh-CN.md) · [繁體中文](README.zh-TW.md) · [日本語](README.ja.md) · **한국어**

**Nemuri는 Claude Code나 Codex가 일하는 동안 — 덮개를 닫아도 — Mac을 깨워 두고, 실행이 끝나면 다시 잠들게 하는 macOS 메뉴 막대 앱입니다.**

에이전트에 긴 작업을 맡기고, 노트북을 덮고, 자리를 뜨세요. 덮개가 닫히는 순간 멈추는 대신 계속 돌아갑니다. 마지막 에이전트가 끝나면 Mac은 알아서 다시 잠듭니다 — 한 시간 전에 끝난 작업 때문에 닫힌 MacBook이 가방 속에서 배터리를 태우는 일이 없습니다.

![Nemuri 메뉴 막대 패널: 에이전트 2개 추적 — 하나는 실행 중, 하나는 확인 대기 — 끝나면 자동 슬립](docs/panel-pro.png)

## 다운로드

- **DMG** — [서명·공증된 최신 릴리스 받기](https://github.com/syfssb/nemuri/releases/latest) (macOS 13+, Apple Silicon & Intel)
- **Homebrew** — `brew install --cask syfssb/nemuri/nemuri`

Mac App Store에는 없습니다 — 진짜 클램셸 모드에 필요한 권한 헬퍼를 샌드박스가 금지하기 때문입니다. 제품 전체 설명은 <https://nemuri.app/ko/>에 있습니다.

## 이런 걸 합니다

- **덮개를 닫고, 외장 디스플레이 없이, 계속 실행.** `caffeinate`만으로는 안 되는, 펌웨어 수준의 `disablesleep` 플래그를 쓰는 진짜 클램셸 모드입니다.
- **실행 중·입력 대기·완료를 구분.** 에이전트가 실제로 일하는 동안만 깨어 있고, 하나가 멈춰 당신의 입력을 기다리는 순간 알려 주며, 마지막 하나가 끝나고 몇 분 뒤 슬립을 되돌려 놓습니다. *(Pro)*
- **세션 패널.** 어떤 에이전트가, 어떤 프로젝트에서, 얼마나 오래 돌고 있는지. *(Pro)*
- **슬립 차단 상태로 절대 걸리지 않음.** 크래시·재부팅·종료·제거, 어느 쪽이든 `disablesleep`은 반드시 `0`으로 돌아갑니다. 복구 경로 4가지, 각각 자동 테스트가 붙어 있습니다.
- **조용함.** 계정 없음, 텔레메트리 없음, 백그라운드 네트워크 없음.

## 이 저장소를 공개하는 이유

Nemuri는 당신에게 **루트 헬퍼** 승인을 부탁드립니다 — root로 실행되며 시스템의 `disablesleep` 플래그를 바꾸는 백그라운드 프로세스입니다. 큰 부탁입니다. 이 저장소는 Nemuri에서 신뢰에 직결되는 부분이며, 우리 말을 믿는 대신 직접 읽어 볼 수 있도록 공개합니다.

**Free와 Pro는 이 저장소의 같은 코드를 실행합니다.** Pro를 사든 사지 않든, 당신의 신뢰가 필요한 부분은 여기 공개된 바로 이 부분입니다.

| 이 저장소에 있는 것 | 무엇인가 |
|---|---|
| `Sources/Helper` | 루트 LaunchDaemon — root로 실행되는 유일한 구성 요소입니다. |
| `Sources/Shared` | 앱과 헬퍼 사이의 XPC 계약 — 헬퍼의 공격 표면 전부입니다. |
| `Sources/Core` | `pmset`/센티널/배터리 프리미티브(절대 걸리지 않는다는 보장), 에이전트 설정을 고쳐 쓰는 인스톨러, 로컬 IPC 프로토콜. |
| `Sources/HookBridge` | `aw-hook` / `aw-codex` — 당신의 `~/.claude/settings.json`과 `~/.codex/config.toml`에 기록되는 아주 작은 바이너리입니다. |

**여기 없는 것:** 감지 엔진, 상태 머신, 오프라인 라이선스 검증, SwiftUI 앱. 모두 비공개 소스이며 — Nemuri Pro가 파는 것이 바로 그 부분입니다. 이 트리만으로 앱 전체를 빌드할 수는 없고, DMG 안의 바이너리도 이 소스만으로 만들어진 것이 아닙니다. 흐리게 두느니 분명히 밝히는 실제 한계입니다.

## 직접 확인할 수 있는 것

- **최소한의 root 공격 표면.** 헬퍼가 노출하는 XPC 메서드는 정확히 세 개 — `setSleepDisabled(Bool)`, `currentState()`, `ping()` — 이고, 그다음엔 `/usr/bin/pmset`을 실행할 뿐입니다. 임의 명령 실행 없음, 센티널 밖으로의 파일 쓰기 없음, 네트워크 없음.
- **절대 걸리지 않음.** 앱이 크래시하든, kill되든, 제거되든 `pmset disablesleep`은 `0`으로 돌아갑니다. 워치독, 60초 센티널 자체 점검, 부팅 시 복구 — 전부 `Sources/Helper/main.swift`에 있습니다.
- **숨은 네트워크 없음.** 이 트리에서 `URLSession`, `Network`, `socket(`, `connect(`를 grep해 보세요 — 여기 있는 소켓은 `AF_UNIX`(로컬 IPC)뿐이고, 루트 헬퍼는 인터넷에 나가지 않습니다.
- **설정은 백업되고 되돌릴 수 있음.** `~/.claude/settings.json`과 `~/.codex/config.toml`을 고쳐 쓰는 코드가 `Sources/Core/Installer.swift`입니다. 쓰기 전에 백업하고, 완전히 복원할 수 있습니다.

## 프라이버시

Nemuri는 평소 운행에서 네트워크를 전혀 쓰지 않습니다. 인터넷에 닿는 건 당신이 직접 일으킬 때뿐입니다: 「업데이트 확인…」 뒤의 업데이트 확인, 그리고 Pro를 구매할 때 이 Mac을 라이선스에 연결하는 한 번의 활성화. 그다음부터 라이선스 검증은 내장된 Ed25519 공개 키로 완전히 로컬에서 이뤄집니다 — 텔레메트리 없음, 백그라운드 재접속 없음. 기기 연결 도입 전에 구매한 라이선스는 그 한 번조차 없이 완전 오프라인으로 활성화됩니다.

## 직접 빌드하기

```bash
swift build   # 루트 헬퍼, XPC 계약, core 프리미티브, hook 브리지 두 개를 빌드합니다
```

macOS 13+와 Swift 5.9+가 필요합니다. 이 트리는 단독으로 빌드됩니다 — 비공개 소스 의존성도, 서드파티 패키지도, 빌드 시 네트워크 접근도 없습니다. root 구성 요소부터 읽고 싶다면 `Sources/Helper/main.swift`(일부러 짧게 썼습니다)와 `Sources/Shared/AwakeShared.swift`(누가 헬퍼에 명령할 자격이 있는지를 정하는 XPC requirement 문자열)에서 시작하세요.
